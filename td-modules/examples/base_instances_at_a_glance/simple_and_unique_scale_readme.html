Simple Instances and Unique Scale

If we can assign scale to one instance, you might already be thinking that we should be able to provide a unique scale value to each instance. That is, in fact, the case. In this example we can that in action.

Open the network and notice that we use a Pattern CHOP with three unique channels - 'sx', 'sy', and 'sz' to provide scale information. For consistency, we use a simple expression to ensure that we have the same number of samples in our patttern CHOP as are in our sopto CHOP. The Length parameter uses the following expression:

op('sopto1').numSamples

This grabs the number of samples from our sopto CHOP to ensure that our CHOPs have matching lengths. This is a common error for those new to using instancing, and it is important to ensure that if you are using CHOPs that you are mindful of the number of samples in your channels.
